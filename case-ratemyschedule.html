RateMySchedule

Winner, Best Pitt-Inspired Demo powered by Gemini API at SteelHacks XI — an AI-powered tool that scores a class schedule on walking time, professor quality, and credit load.

teamAidan McHugh, Vaishu Kusumba, Lena Carson, James Toscano
roleFull-Stack Developer
skillsReact, Flask, Python, Gemini API
ambitionTake the dread out of registration week.
the problem

How do you know if your class schedule is actually good, not just technically possible?

the process

We built RateMySchedule around one input: the .ics file every Pitt student can download straight from PeopleSoft. Once uploaded, we convert it to a .csv and parse out each class along with the breaks between them. For each break, we needed real walking times — since we didn't have access to the Google Maps API, we manually collected coordinates for every Pitt academic building and ran them through the HERE Routing API to calculate actual walking distances.

For professor quality we pulled ratings, difficulty, and "would take again" percentages through a RateMyProfessor API, and for credit load we compared each student's total against typical course loads. All three scores then fed into the Gemini API, which we prompted to turn raw numbers into a readable, encouraging report card instead of three cold statistics.

the solution

A student downloads their schedule, uploads it, and enters their professors and credit counts for each class. RateMySchedule scores the schedule across three categories — break feasibility, professor quality, and credit load — and Gemini writes a personalized report card explaining the reasoning behind each grade, in a tone that's encouraging rather than clinical.

what i learned

This was the first full-stack, API-heavy project for all four of us, so a lot of the learning was just getting comfortable — with Flask, with React, with reading API docs under a 24-hour clock. The more interesting lesson was in prompting Gemini itself: getting it to sound genuinely helpful instead of generic took several rounds of rewriting the prompt, which taught me that "AI-powered" features need just as much design attention as any other part of the interface.
